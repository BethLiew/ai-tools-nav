# 部署指南

## 快速部署到 GitHub Pages

### 步骤 1：在 GitHub 创建仓库

1. 登录 GitHub：https://github.com
2. 点击右上角 "+" > "New repository"
3. 仓库名称：`ai-tools-nav`
4. 选择 "Private" 或 "Public"
5. 点击 "Create repository"

### 步骤 2：推送代码

在本地终端执行以下命令（替换 YOUR_USERNAME 为你的GitHub用户名）：

```bash
cd AI导航站/ai-tools-nav

# 添加远程仓库
git remote add origin https://github.com/YOUR_USERNAME/ai-tools-nav.git

# 推送代码
git push -u origin main
```

### 步骤 3：启用 GitHub Pages

1. 进入仓库页面
2. 点击 "Settings"（设置）
3. 左侧菜单找到 "Pages"
4. Build and deployment:
   - Source: 选择 "Deploy from a branch"
   - Branch: 选择 "gh-pages" / "/" (或者选择 GitHub Actions)
5. 点击 "Save"

### 步骤 4：等待部署

- GitHub Actions 会自动构建并部署
- 大约 1-2 分钟后站点即可访问
- 访问地址：`https://YOUR_USERNAME.github.io/ai-tools-nav/`

## 购买并绑定自定义域名（可选）

### 购买域名
推荐域名注册商：
- Namecheap（价格便宜）
- GoDaddy（老牌）
- 阿里云（国内）
- 腾讯云（国内）

推荐域名：
- ai-tools.cn（便宜）
- aitools.fun
- ai-nav.com

### 配置域名

1. 在域名控制台添加 DNS 记录：
   - A 记录：`185.199.108.153`（指向 GitHub Pages IP）
   - CNAME：`YOUR_USERNAME.github.io`

2. 在 GitHub 仓库 Settings > Pages 添加自定义域名

3. 等待 DNS 生效（通常几分钟到24小时）

## 使用 Vercel 部署（替代方案）

如果 GitHub Pages 访问慢，可以使用 Vercel：

1. 注册 Vercel：https://vercel.com
2. 导入 GitHub 仓库
3. 框架选择 "Hugo"
4. 点击 Deploy

## 使用 Netlify 部署（替代方案）

1. 注册 Netlify：https://netlify.com
2. 连接 GitHub 仓库
3. 构建命令：`hugo`
4. 发布目录：`public`

## 本地开发

```bash
# 启动本地服务器
cd AI导航站/ai-tools-nav
hugo server

# 访问 http://localhost:1313

# 构建生产版本
hugo --minify
```

## 常见问题

### Q: 部署后样式不显示？
A: 检查 baseURL 配置是否正确，确保末尾有斜杠

### Q: 中文显示乱码？
A: 确保 config.toml 中 languageCode 为 "zh-cn"

### Q: 页面404？
A: 检查 GitHub Pages 的 Source 设置，或等待几分钟让部署完成

### Q: 如何更新内容？
A: 修改 content/ 下的 .md 文件，然后推送代码即可自动更新
